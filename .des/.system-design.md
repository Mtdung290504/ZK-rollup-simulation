# Thiết kế Hệ thống: ZK-Rollup Simulator (PoC)

Tài liệu này quy định cấu trúc kỹ thuật, luồng dữ liệu và thiết kế trạng thái cho hệ thống mô phỏng ZK-Rollup.
Thiết kế sử dụng cơ chế **"Treasury Symmetric Bridge"** để tối ưu hóa mạch ZK tĩnh.

---

## 1. Tổng quan Kiến trúc (Architecture Overview)

Hệ thống phân tán thành **4 thành phần độc lập**, giao tiếp qua HTTP/JSON:

1. **Layer 1 Server (Port 3000)**
    - Mô phỏng Ethereum Smart Contract
    - Giữ ETH thật
    - Xác thực ZK Proof
    - Chỉ lưu trữ các mã băm (Roots) để tối ưu chi phí

2. **Layer 2 Server (Port 5000)**
    - Mô phỏng Sequencer Node
    - Xử lý logic Soft Finality
    - Quản lý Mempool
    - Cập nhật State Tree
    - Gọi SnarkJS để sinh Proof

3. **Frontend App (Port 8080)**
    - Ví người dùng
    - Quản lý cặp khóa EdDSA
    - Giao diện Bridge (Nạp/Rút)

4. **DA Observer Tool (CLI/Script)**
    - Công cụ off-chain
    - Tải Data Blob
    - Dựng Merkle Tree DA
    - Cấp Merkle Proof phục vụ rút tiền

## 3. Luồng Dữ liệu Cốt lõi (Core Data Flows)

### 3.1. Luồng Nạp tiền (Deposit: L1 → L2)

- User gọi POST /rpc/contract/ZkRollup/deposit → L1 khóa ETH và ghi pending_deposits
- Sequencer đọc event → Treasury ký tx [Treasury → User]
- Tx vào batch → ZK xử lý → cập nhật state
- Quan hệ kế toán:
  total_locked_eth = MAX_UINT128 - treasury_balance

---

### 3.2. Luồng Chuyển tiền (Transfer: L2 → L2)

- POST /api/l2/transfer
- Sequencer validate FIFO
- Update state ngay
- Append vào transactions[]

---

### 3.3. Luồng Rút tiền (Withdraw: L2 → L1)

**Giai đoạn 1:**

- User → Treasury (burn logic)
- Batch + proof → submit L1
- L1 lưu state root + DA root

**Giai đoạn 2:**

- User dựng Merkle proof từ Blob
- Gọi withdraw
- L1 verify → trả ETH → mark nullifier

## 4. API Interfaces

### L1:

- POST /rpc/contract/ZkRollup/deposit
- POST /rpc/contract/ZkRollup/submitBatch
- POST /rpc/contract/ZkRollup/withdraw
- GET /api/archive/blobs/:batch_id

### L2:

- POST /api/l2/transfer
- GET /api/l2/sync-l1
- POST /api/l2/force-batch

## 5. Bảo mật

- No inflation: không mint trong circuit
- Data availability: dùng DA root
- Safety: proof từ L1
- Liveness: batch + padding
